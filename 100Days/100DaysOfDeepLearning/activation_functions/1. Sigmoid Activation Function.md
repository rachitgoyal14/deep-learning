# Sigmoid Activation Function

## Definition

The sigmoid activation function (also called the logistic function) is defined as:

$$\sigma(x) = \frac{1}{1 + e^{-x}}$$

## Sigmoid and Its Derivative

![[Pasted image 20260212014458.png]]

In the above picture:

- **First graph:** Sigmoid function $\sigma(x)$ - outputs range from 0 to 1
- **Second graph:** Derivative of sigmoid $\sigma'(x)$ - has a **peak value of 0.25** and only has some decent value between **-3 and 3**, and beyond that, it approaches zero

The derivative of sigmoid is given by: $$\sigma'(x) = \sigma(x)(1 - \sigma(x))$$

**Key observation:** The maximum gradient is 0.25, which occurs at $x = 0$. For values of $|x| > 3$, the gradient becomes very close to zero.

---

## Advantages of Sigmoid Function

### 1. **Probabilistic Interpretation**

- The output of sigmoid which is between 0 and 1 can be treated as a **probability** and is often done so while dealing with a binary classification problem
- This is the reason often in the binary classification problem, the activation function in the output layer is sigmoid
- Maps any real-valued number to a probability-like value
- Example: Output of 0.8 can be interpreted as 80% confidence in class 1

### 2. **Non-Linear Function**

- It's a non-linear function, which means it can capture the patterns of non-linear data
- Enables neural networks to learn complex decision boundaries
- Without non-linearity, multiple layers would collapse into a single linear transformation

### 3. **Differentiable**

- It is differentiable everywhere, which helps in backpropagation and gradient descent
- Smooth gradient allows for stable optimization
- Derivative has a simple closed form: $\sigma'(x) = \sigma(x)(1 - \sigma(x))$

---

## Disadvantages of Sigmoid Function

### 1. **Vanishing Gradient Problem (Saturating Function)**

It's a **saturating function** - even for $-\infty$ to $+\infty$ input, the output is always between **0 and 1**, which can lead to the **vanishing gradient problem**.

**Why this happens:**

Since the formula of sigmoid is: $$\sigma(x) = \frac{1}{1 + e^{-x}}$$

The derivative is: $$\sigma'(x) = \sigma(x)(1 - \sigma(x))$$

**Key issue:** $\sigma'(x) \leq 0.25$ for all values of $x$

**During backpropagation:**

- Gradients are multiplied layer by layer: $\frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial a_n} \cdot \sigma'(z_{n-1}) \cdot \sigma'(z_{n-2}) \cdot \ldots \cdot \sigma'(z_1)$
- Each multiplication by $\sigma'(x)$ shrinks the gradient by a factor $\leq 0.25$
- In deep networks: $0.25^{10} = 0.0000009537$ - gradient vanishes exponentially!
- Early layers receive almost zero gradient → weights don't update → network fails to learn

**This is the biggest reason why sigmoid is not used in hidden layers but only in output layers these days.**

---

### 2. **Non-Zero Centered Output**

Sigmoid activation function is **not zero-centered**. When the activation sum is calculated after sigmoid, that number is not normalized - basically the **mean is not zero**, it's either positive or negative. This leads to **slower training** which means difficulty in convergence.

#### **Intuitive Mathematical Explanation:**

Consider a simple neuron: $$y = \sigma(w_1x_1 + w_2x_2 + b)$$

During backpropagation, we need to update weights using: $$\frac{\partial L}{\partial w_i} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial w_i}$$

where: $$\frac{\partial y}{\partial w_i} = \sigma'(z) \cdot x_i$$

**The Problem:**

- Sigmoid outputs are always **positive** (range: 0 to 1)
- This means all $x_i$ values fed to the next layer are positive
- The gradient with respect to weights becomes: $$\frac{\partial L}{\partial w_i} = \frac{\partial L}{\partial y} \cdot \sigma'(z) \cdot x_i$$

**Critical Issue:** Since all $x_i > 0$ (sigmoid outputs), the sign of $\frac{\partial L}{\partial w_i}$ is determined **entirely** by $\frac{\partial L}{\partial y}$

This means:

- **All weights must move in the same direction** (all increase or all decrease) in a given iteration
- Even if the optimal update requires some weights to increase and others to decrease
- This constraint severely limits the optimization path

#### **The Zig-Zag Motion:**

Imagine the optimal weight update is:

- $w_1$ should increase by +2
- $w_2$ should decrease by -3

But because all inputs are positive (from sigmoid), gradients force:

- **Iteration 1:** Both $w_1$ and $w_2$ increase (say +1 each)
- **Iteration 2:** Both $w_1$ and $w_2$ decrease (say -1 each)
- **Iteration 3:** Both increase again...

**Result:** The optimization path follows a **zig-zag pattern** instead of taking the direct path to the optimum.

```
      Optimal Path: Direct diagonal
            ↓
    w₂ |     X (optimum)
       |    /
       |   /
       |  / ← Direct path
       | /
       |/___________
       O        w₁
       
    Actual Path: Zig-zag
            ↓
    w₂ |     X (optimum)
       |    /|
       |   / |
       |  /  |\
       | /   | \
       |/____↗__↘___
       O        w₁
       ↑
    Zig-zag motion due to
    non-zero-centered activations
```

**Why zero-centered helps:**

- Functions like **tanh** output range [-1, 1], with mean ≈ 0
- Inputs can be positive or negative
- Weights can update independently in different directions
- Faster convergence along the optimal path

---

### 3. **Computationally Expensive**

Since the formula of sigmoid is: $$\sigma(x) = \frac{1}{1 + e^{-x}}$$

It involves:

- **Exponential computation** ($e^{-x}$) - expensive operation
- Division operation
- Multiple floating-point operations per neuron

**In a neural network:**

- Millions of neurons
- Each neuron computes sigmoid during forward pass
- Each neuron computes sigmoid derivative during backward pass
- This exponential computation happens millions of times per training iteration

**Comparison:**

- **ReLU:** $\max(0, x)$ - just a simple comparison and selection
- **Sigmoid:** $\frac{1}{1 + e^{-x}}$ - exponential, addition, division

This computational cost significantly increases training time, especially for large-scale deep networks.

**This is the reason why sigmoid function is not used these days in the hidden layers. It's only used in the output layer while dealing with a binary classification problem.**

## Summary

|Aspect|Sigmoid|
|---|---|
|**Formula**|$\sigma(x) = \frac{1}{1 + e^{-x}}$|
|**Range**|$(0, 1)$|
|**Zero-centered**|✗ No|
|**Saturating**|✓ Yes (both ends)|
|**Max Gradient**|0.25|
|**Computational Cost**|High (exponential)|
|**Vanishing Gradient**|✓ Severe problem|
|**Current Usage**|Output layer for binary classification only|
|**Hidden Layers**|✗ Not recommended (use ReLU instead)|

---

## Modern Usage

**When to use Sigmoid:**

- ✅ Binary classification output layer
- ✅ When you need probabilistic interpretation (0 to 1)
- ✅ Gates in LSTM/GRU cells (input, forget, output gates)

**When NOT to use Sigmoid:**

- ❌ Hidden layers in deep networks (use ReLU, Leaky ReLU, or ELU)
- ❌ When you need zero-centered outputs (use tanh)
- ❌ When computational efficiency is critical
- ❌ Very deep networks (vanishing gradient kills learning)
