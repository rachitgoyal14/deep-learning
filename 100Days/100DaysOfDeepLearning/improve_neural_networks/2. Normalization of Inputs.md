## The Problem: Different Scales

Feature scaling is used when the **range/scale of data varies significantly** across different features.

### Real-World Example: Predicting Salary from Age

**Scenario:** Training a neural network to predict salary based on age

**The Scale Problem:**

- **Age:** Ranges from 0 to 70 (small scale)
- **Salary:** Ranges from thousands to lakhs to crores (huge scale)

```
Age:     [25,  30,  45,  60]
Salary:  [300000, 500000, 1500000, 8000000]

Scale difference: ~100,000x !
```

**Consequence:** Neural network takes **a lot of time to learn** - weights must compensate for vastly different scales.

---

## Understanding Through Backpropagation

### The Update Formula

$$w_{new} = w_{old} - \eta \cdot \frac{\partial L}{\partial w}$$

### Feature-Specific Weights

Consider two features:

- $$x_1$$ → Age (range: 0-70)
- $$x_2$$ → Salary (range: 0-10,000,000)

With corresponding weights:

- $$w_1$$ → Weight for age
- $$w_2$$ → Weight for salary

### The Problem in Updates

**Gradient calculation:**

$$\frac{\partial L}{\partial w_1} = \frac{\partial L}{\partial \hat{y}} \cdot x_1 \quad (\text{involves age: } \sim 50)$$

$$\frac{\partial L}{\partial w_2} = \frac{\partial L}{\partial \hat{y}} \cdot x_2 \quad (\text{involves salary: } \sim 5,000,000)$$

**Weight updates:**

$$w_1 \leftarrow w_1 - \eta \cdot (\text{small gradient})$$

$$w_2 \leftarrow w_2 - \eta \cdot (\text{huge gradient})$$

**Result:**

- $$w_2$$ changes drastically (overshoots)
- $$w_1$$ barely changes (undershoots)
- Training is **unstable and slow**

```
Epoch 1:  Overshoots → High loss
Epoch 2:  Corrects → Overshoots other way
Epoch 3:  Oscillates back

```

### Normalized Data (Right Side)

**Cost Function Shape:** Circular, symmetric

```
        w2
        ↑
    ____|____
   /    |    \
  |     |     |
  |     *     |  ← Optimal point
  |           |
   \____↓____/
        |
        → w1

Circular contour
```

**Characteristics:**

- Symmetric contours (circular)
- Equal scale in all directions
- Gradient descent takes **direct path** to minimum
- Fast convergence

**Training Behavior:**

```
Epoch 1:  Step toward minimum
Epoch 2:  Step closer
Epoch 3:  Step closer
...
Epoch 20: Converged!
```

**From Andrew Ng's Deep Learning Specialization:**

> "If the features are on very different scales, the cost function will be elongated. Gradient descent will oscillate inefficiently down to the minimum. Feature scaling makes the cost function more symmetric, allowing faster convergence."

## Feature Scaling Techniques

### 1. Standardization (Z-Score Normalization)

**Formula:**

$$x_{scaled} = \frac{x - \mu}{\sigma}$$

Where:

- $$\mu$$ = mean of feature
- $$\sigma$$ = standard deviation of feature

**Effect:**

- Centers data around 0
- Scales to unit variance
- Result: Data roughly in range $$(-3, +3)$$, mostly $$(-1, +1)$$

**When to Use:**

- Data is **normally distributed** (bell curve)
- Features have different units (age in years, salary in rupees)
- Want to preserve shape of distribution

**Example:**

```
Original: [20, 30, 40, 50, 60]
Mean: 40, Std: 14.14

Standardized:
[20-40]/14.14 = -1.41
[30-40]/14.14 = -0.71
[40-40]/14.14 =  0.00
[50-40]/14.14 =  0.71
[60-40]/14.14 =  1.41

Result: [-1.41, -0.71, 0, 0.71, 1.41]
```

**Visualization:**

![[Pasted image 20260205102246.png]]

**Properties:**

- Mean = 0
- Standard deviation = 1
- Handles outliers better (doesn't squash them to extremes)
- Most common choice for neural networks

---

### 2. Normalization (Min-Max Scaling)

**Formula:**

$$x_{scaled} = \frac{x - x_{min}}{x_{max} - x_{min}}$$

**Effect:**

- Scales data to fixed range $$[0, 1]$$
- Preserves zero entries
- Bounded output

**When to Use:**

- When you need **bounded features** (0 to 1)
- Data is **NOT normally distributed**
- When there are **outliers** (but be cautious!)

**Example:**

```
Original: [20, 30, 40, 50, 60]
Min: 20, Max: 60

Normalized:
(20-20)/(60-20) = 0.00
(30-20)/(60-20) = 0.25
(40-20)/(60-20) = 0.50
(50-20)/(60-20) = 0.75
(60-20)/(60-20) = 1.00

Result: [0, 0.25, 0.5, 0.75, 1.0]
```

**Caution with Outliers:** If you have an outlier (e.g., 1000), it will squash all other values near 0!

## Sklearn Implementation

### StandardScaler (Standardization)

**How it works internally:**

```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()

# Step 1: fit() - Calculate mean and std from training data
scaler.fit(X_train)
# Stores: scaler.mean_, scaler.scale_ (std)

# Step 2: transform() - Apply the scaling
X_train_scaled = scaler.transform(X_train)
# Formula: (X - mean) / std

# Or combine both:
X_train_scaled = scaler.fit_transform(X_train)

# Step 3: Transform test data using SAME mean and std
X_test_scaled = scaler.transform(X_test)
```

**Critical:** Always `fit()` on training data only, then `transform()` both train and test!

**Internal Attributes:**

- `scaler.mean_`: Mean of each feature from training data
- `scaler.scale_`: Standard deviation of each feature from training data

---

### MinMaxScaler (Normalization)

**How it works internally:**

```python
from sklearn.preprocessing import MinMaxScaler

scaler = MinMaxScaler()

# Step 1: fit() - Calculate min and max from training data
scaler.fit(X_train)
# Stores: scaler.data_min_, scaler.data_max_

# Step 2: transform() - Apply the scaling
X_train_scaled = scaler.transform(X_train)
# Formula: (X - min) / (max - min)

# Or combine:
X_train_scaled = scaler.fit_transform(X_train)

# Transform test data using SAME min and max
X_test_scaled = scaler.transform(X_test)
```

**Internal Attributes:**

- `scaler.data_min_`: Min of each feature from training data
- `scaler.data_max_`: Max of each feature from training data

## Best Practices

### 1. Always Scale in Deep Learning

**Rule of thumb:** If features have different scales, ALWAYS scale!

**When it's critical:**

- Features with different units (age in years, salary in rupees)
- Features with vastly different ranges
- Using gradient-based optimizers

### 2. Fit on Training Data Only

```python
# CORRECT
scaler.fit(X_train)
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)

# WRONG - Data leakage!
scaler.fit(X)  # Don't fit on entire dataset
```

**Why?** Test data should be truly unseen - using its statistics is cheating!

### 3. StandardScaler for Most Cases

**Preferred choice:** StandardScaler (standardization)

**Reasons:**

- Works well with normal distributions
- Handles outliers better than MinMaxScaler
- Most compatible with neural network assumptions
- Doesn't bound values (unlike MinMax)

**Use MinMaxScaler when:**

- You specifically need [0, 1] range
- Data is uniformly distributed
- Working with image data (already 0-255)

### 4. Save the Scaler

```python
import pickle

# Save scaler for deployment
with open('scaler.pkl', 'wb') as f:
    pickle.dump(scaler, f)

# Load and use later
with open('scaler.pkl', 'rb') as f:
    scaler = pickle.load(f)
    new_data_scaled = scaler.transform(new_data)
```

**Why?** In production, you need the same scaling for new predictions!

## Common Mistakes

### 1. Forgetting to Scale Test Data

```python
# Wrong!
X_train_scaled = scaler.fit_transform(X_train)
model.fit(X_train_scaled, y_train)
predictions = model.predict(X_test)  # Using unscaled test data!
```

**Fix:** Always scale test data with same scaler!

### 2. Fitting Scaler on Test Data

```python
# Wrong!
X_test_scaled = scaler.fit_transform(X_test)  # Don't fit on test!
```

**Fix:** Only `transform()` test data, never `fit()`!

### 3. Scaling Target Variable Unnecessarily

```python
# Usually not needed for classification
y_scaled = scaler.fit_transform(y)  # Don't scale 0/1 labels!
```

**When to scale y:**

- Regression with large target values (yes, scale)
- Classification (no, keep as 0/1 or one-hot)

### 4. Not Scaling in Production

**Training:** Use scaled data **Production:** Forget to scale → Model fails!

**Solution:** Always apply same scaler to new data before prediction.

---

**Always scale your features before training neural networks!** It's one of the simplest yet most impactful preprocessing steps.

**From Goodfellow's "Deep Learning":**

> "Feature scaling is not just a nice-to-have preprocessing step - it's often essential for gradient-based optimization to work effectively. The geometry of the optimization landscape changes dramatically with proper scaling."
